Operadores Relacionais

Voltar

Operadores relacionais em JavaScript são usados ​​para comparar dois valores e retornar um valor booleano ( `true` ou `false`) com base no resultado da comparação. Aqui estão os operadores relacionais mais comumente usados ​​em JavaScript:

  • Igual a (==): compara dois valores quanto à igualdade, após convertê-los em um tipo comum.

    • let a = 5;

    let b = '5';

    console.log(a == b);

    Observe que o == operador às vezes pode levar a resultados inesperados devido à coerção de tipo, portanto, geralmente é recomendado usar o operador de igualdade estrita (===).

  • Estritamente igual a (===): compara dois valores quanto à igualdade sem coerção de tipo.

    • let a = 5;

    let b = '5';

    console.log(a === b);

  • Diferente de (!=): compara dois valores quanto à desigualdade, após convertê-los para um tipo comum.

    • let a = 5;

    let b = '5';

    console.log(a != b);

    Novamente, é recomendado usar o operador estrito !== diferente de ( ).

  • Estritamente diferente de (!==): Compara dois valores para desigualdade sem coerção de tipo.

    • let a = 5;

    let b = '5';

    console.log(a !== b);

  • Maior que (>): Retorna truese o operando esquerdo for maior que o operando direito.

    • let a = 5;

    let b = '5';

    console.log(a > b);

  • Menor que (<) : Retorna truese o operando esquerdo for menor que o operando direito.

    • let a = 5;

    let b = '5';

    console.log(a < b);

  • Maior ou igual a (>=): Retorna truese o operando esquerdo for maior ou igual ao operando direito.

    • let a = 5;

    let b = '5';

    console.log(a >= b);

  • Menor ou igual a (<=) : Retorna truese o operando esquerdo for menor ou igual ao operando direito.

    • let a = 5;

    let b = '5';

    console.log(a <= b);

    É importante observar que ao usar operadores relacionais com valores não numéricos, o JavaScript tentará primeiro convertê-los em números. No entanto, isso às vezes pode levar a resultados inesperados, por isso geralmente é uma boa ideia certificar-se de que você está trabalhando explicitamente com números.
