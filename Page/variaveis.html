Características de Tipagem

Primeiramente, devemos citar que a linguagem JavaScript é de tipagem dinâmica, ou seja, não é necessário declarar tipo a uma variável, podendo ser modificada posteriormente.

A própria linguagem de programação, neste caso, tem a habilidade de escolher o tipo de dado de acordo com o valor atribuído a variável.

Tipos de dados

Ao programar em JavaScrpit nós precisamos saber quais os tipos de dados que a linguagem trabalha, os principais tipos de dados primitivos são:

  • Boolean - possuem apenas dois valores: verdadeiro ou falso;
  • Undefined - indica que não foi definido um valor;
  • Null - indica que um valor é nulo;
  • Number - armazena valores numéricos;
  • String - armazena textos;
  • Symbol - armazena símbolos;

Além dos tipos de dados primitivos, nós também temos o tipo object, que possibilita trabalharmos com objetos utilizando JavaScript.

Regras de nomeação de variáveis

Uma das peculiaridades da linguagem JavaScript é que podemos declarar variáveis de duas formas, utilizando var, comum em outras linguagens, ou utilizando let.

Vale ressaltar que por se tratar de uma linguagem de tipagem dinâmica, ao declarar uma variável não é necessário informar o tipo da variável, por exemplo:

var idade = 16;

var nome = 'João';

var endereco = 'Av. Central 1';

Ao atribuir o valor 15 para a variável idade, a linguagem já reconhece como uma variável do tipo Number. O mesmo ocorre para a variável nome, ao atribuirmos o valor João a linguagem reconhece como uma variável do tipo String.

Outras regras devem ser seguidas, como:

  • Não declarar variável com palavras reservadas, como let, var, while, for, etc.
  • Deve conter somente letras, números e underscore.
  • Pode iniciar com uma letra, underscore (_) ou cifrão ($).

Declarando variáveis com var e let

No JavaScript nós temos que nos atentar quando vamos declarar variáveis usando var ou let, pois cada uma possuí diferenças que são importantes.

Antes de tudo precisamos explicar que o JavaScript trabalha com escopos, onde cada bloco (funções, condicionais e estruturas de repetição, por exemplo) é um escopo fechado.

Ao utilizarmos var, ele trabalha de forma independente para cada escopo de função, por exemplo:

var nomeCarro = 'Ford F-100';

console.log(nomeCarro);

var nomeCarro = 'Dodge D100';

console.log(nomeCarro);

O usuário forneceu um pequeno trecho de código que declara uma variável chamada `nomeCarro` e atribui a ela o valor da string "Ford F-100". Em seguida, ele registra o valor `nomeCarro` no console, altera o valor para "Dodge D100" e o registra novamente no console.

Agora se subtituirmos o código acima declaramos as variaveis como let:

let nomeCarro = 'Ford F-100';

console.log(nomeCarro);

let nomeCarro = 'Dodge D100';

console.log(nomeCarro);

O usuário forneceu outro trecho de código JavaScript que declara uma variável chamada `nomeCarro` e atribui a ela o valor da string "Ford F-100". Em seguida, ele registra o valor `nomeCarro` no console, altera o valor para "Dodge D100" e o registra novamente no console. No entanto, há uma pequena diferença entre este código e o anterior: desta vez, ele está usando a `let` palavra-chave em vez de `var`.

No entanto, é importante notar que a redeclaração de variáveis ​​às vezes pode dificultar a leitura e a compreensão do seu código. Neste exemplo específico, você poderia simplesmente atribuir um novo valor à `nomeCarro` sem declará-la novamente, assim:

let nomeCarro = 'Ford F-100';

console.log(nomeCarro);

nomeCarro = 'Dodge D100';

console.log(nomeCarro);

Dessa forma, você evita declarar novamente a variável e deixa seu código um pouco mais claro.

Declarando constantes

Assim como em outras linguagens, podemos declarar constantes que possuem valores fixos. Também vale ressaltar que as constantes tem o mesmo comportamento que variáveis definidas como let em relação ao escopo de bloco, como vimos no exemplo acima.

const PI = 3.14;

Mostrar outro exemplo.

O usuário forneceu outro trecho de código JavaScript que declara duas variáveis, `endereco` e `nome`, e registra seus valores no console duas vezes. No entanto, desta vez, ele está usando uma combinação das palavras-chave `const` e `var`, e está tentando declarar novamente a `endereco` variável com um novo valor.

const endereco = 'Av. Central 1';

var nome = 'João Pedro';

console.log(endereco,nome);

const endereco = 'Av. Central 2';

var nome = 'Alemão';

console.log(endereco,nome);

Este código contém alguns problemas. Primeiro, está tentando declarar novamente a `endereco` constante com um novo valor, mas isso não é permitido em JavaScript. Depois que uma variável constante é declarada, seu valor não pode ser alterado. Portanto, a segunda declaração de `endereco` um arquivo `SyntaxError`.

Segundo, está tentando declarar novamente a `nome` com um novo valor, mas isso não é necessário. Você pode simplesmente atribuir um novo valor à `nome` existente sem declará-la novamente.

Versão correta:

const endereco = 'Av. Central 1';

let nome = 'João Pedro';

console.log(endereco,nome);

endereco = 'Av. Central 2';

nome = 'Alemão';

console.log(endereco,nome);

Nesta versão corrigida, estamos usando `const` para a `endereco` variável, pois não precisamos alterar seu valor, e `let` para a `nome` variável, pois precisamos alterar seu valor. Também estamos removendo as redeclarações desnecessárias.

Conclusão

Concluindo, o JavaScript é uma linguagem de programação versátil que oferece características de tipagem dinâmica, permitindo a declaração de variáveis sem a necessidade de especificar seu tipo. A linguagem é capaz de inferir o tipo de dado com base no valor atribuído à variável. Além dos tipos de dados primitivos, como boolean, undefined, null, number, string e symbol, o JavaScript também suporta objetos, o que amplia suas capacidades de programação.

As regras de nomeação de variáveis devem ser seguidas para garantir a clareza e a legibilidade do código, incluindo evitar palavras reservadas, utilizar apenas letras, números e underscore, e seguir padrões de nomenclatura adequados.

Ao declarar variáveis, é importante entender as diferenças entre usar var e let. Enquanto var funciona de forma independente para cada escopo de função, let trabalha com escopos de bloco, proporcionando maior controle sobre o fluxo do código e evitando problemas de redeclaração.

Além disso, o JavaScript permite a declaração de constantes, que são variáveis com valores fixos e comportamento semelhante às variáveis definidas com let em relação ao escopo de bloco. É essencial evitar tentativas de redeclarar constantes, pois isso resultará em erros de sintaxe.

Em resumo, ao programar em JavaScript, é crucial compreender suas características de tipagem dinâmica, regras de nomeação de variáveis e diferenças entre var, let e const, a fim de escrever código claro, conciso e sem erros.
